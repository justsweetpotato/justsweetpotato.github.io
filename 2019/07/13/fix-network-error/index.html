解决 PC 端关闭代理软件后无法上网的问题

Posted on 2019-07-13

“什么情况? 电脑不翻墙不能上网了? 这是逆向 GFW?!!!” — 来自肥宅的抱怨.

最近又遇到了这个在我还是 “小白” 时期, 最常遇到的 BUG, 几乎是肌肉记忆般的打开 “局域网(LAN)设置”, 取消勾选 “为 LAN 设置代理服务器”, 啊咧? 问题依然存在, 只好运用最常使用的排除法, 一个个排除所有能考虑到的可能性.

在发生问题后, 我通常会做大量测试来将问题限定在某个具体或大概的范围内, 以便于排查和解决问题.

赶时间的朋友请忽略下方思路, 这里直接贴出结果:

  1. 检查 IE 设置中的 “为 LAN 设置代理服务器”, 确保没有被勾选
  2. 检查 DNS 设置, 可设置为 114.114.114.114, 223.5.5.5, 8.8.8.8
  3. 检查 PAC 文件
  4. 检查 hosts 设置
  5. 重置网卡

问题 1

SwitchyOmega 使用 PAC 模式, 能正常浏览 YouTube, Google, 无法打开 简书, V2EX等, 表现为无限转圈圈.

[分析]
国内网址无法被解析

  • DNS 故障, 翻墙网站会使用远程 DNS 解析, 因为能正常访问被墙网站, 故本地 DNS 故障.
  • 国内网址被导向错误地址
    • PAC 规则错误
    • hosts 文件错误

[解决方案]

  • 检查 PC 端 DNS 设置, 检查路由器 DNS 设置, (PC 端设置优先级最高) 均正常, 但我的 DNS 是 谷歌公共 DNS(存在被干扰可能), 故 ping 114.114.114.114 成功后暂时先将 DNS 设置为 114DNS
  • 开启隐身窗口(浏览器缓存会影响测试结果), 将 SwitchyOmega 设置为直连模式测试国内网站, 失败
  • 检查 SwitchyOmega 在线 PAC 规则, 正常, 重新更新规则, 正常
  • 开启隐身窗口, 直连模式测试国内网站, 失败
  • 检查 hosts 文件, 正常

[结果]
排除 DNS 故障, 排除 PAC 文件, 排除 hosts 文件的问题.

问题 2

关闭翻墙软件后, 墙内墙外网站均无法访问, ping 任意地址均无法 ping 通, nslookup 查询 IP 地址出现 timeout.

[分析]
发生的问题可能并不在翻墙软件本身, 而在于更大的层面, 电脑的网络问题(功能故障[内部], 或网络故障[外部])

  • 翻墙软件异常退出, 致使未自动关闭 “为 LAN 设置代理服务器”, 而代理服务器实际并未开启, 导致无法上网
  • 电脑与网络连接有关的系统故障
  • 网络故障

[解决方案]

  • 检查 IE 设置中 “为 LAN 设置代理服务器” 选项, 确保选项被关闭, 再次打开隐身窗口测试, 失败
  • 使用手机热点测试, 仍然失败
  • 重装系统, 重装是不可能重装的, “修电脑的” 最常做的就是重装系统, 恕我直言 low 爆了, 这根本没有解决问题或是试图去解决问题, 这样的态度是无法接受的

[结果]
排除翻墙软件问题, 排除网络问题, 将问题锁定至电脑系统.

问题 3

开启 - 关闭 - 开启 现在正在使用的网卡, 尝试直接使用 宽带拨号. 均失败.

[分析]
这个问题有点严重, 为什么不问问万能的谷歌呢?

  • 中文搜索结果1: 牛头不对马嘴
  • 中文搜索结果2: 已尝试过, 不行
  • 中文搜搜结果3: 你 TM 是智障吗? 您脑子没问题吗

[解决方案]
只好抱着试一试的心态重置网卡, 因为记得以前安装过网络方面的软件卸载的时候都会提示要你重置网卡(网易 UU, 360 WIFI等)

[结果]
恢复正常!

总结

仍然没找到什么原因导致了这次的故障, 合理怀疑刚刚卸载的 蓝叠模拟器(国际版) 导致网卡故障.

题外话

曾经使用 Shadowsocks 同时使用 迅雷 出现过 1080 端口被占用的情况, 导致 Shadowsocks 无法使用.

据报道, 腾讯旗下游戏 英雄联盟 出现搜集用户 Shadowsocks 配置文件的情况, 巧合的是, 几个月后, 2019 年春节, Shadowsocks 出现大面积被封端口情况, 包括自用线路, 常规端口部署, 虽然换个端口就可以继续用了, 可以肯定的是 Shandowsocks 的流量特征已经能被识别. 这难道是巧合吗?

2019 年 6 月初, 梯子大规模被封禁, 国外知名 VPS 服务器网段被封, 一时之间仿佛回到了蛮荒时代, 用百度? 你在开玩笑吗? 这之后, 网友纷纷使用 TLS(HTTPS) 以及 CDN 转发的技术来应对这次的封锁.

如果你很在意你的信息安全, 请不要使用国内软件, 敏感操作不要使用手机, 手机安全隐患太大.
